# Requirements

Everything you need to run Keyflare.

## For Deployment

To deploy Keyflare to Cloudflare, you need:

| Requirement            | Details                   |
| ---------------------- | ------------------------- |
| **Node.js**            | Version 20 or higher      |
| **Cloudflare Account** | Free tier works perfectly |
| **pnpm** (optional)    | For building from source  |

### Cloudflare Permissions

Your Cloudflare account needs these permissions:

| Permission             | Purpose                             |
| ---------------------- | ----------------------------------- |
| `Workers Scripts:Edit` | Deploy and manage the Worker        |
| `D1:Edit`              | Create and manage the database      |
| `Workers Routes:Edit`  | Configure custom domains (optional) |

## For Local Development

For local development, you only need:

| Requirement | Details              |
| ----------- | -------------------- |
| **Node.js** | Version 20 or higher |
| **pnpm**    | Version 9 or higher  |

No Cloudflare account is required for local development. Everything runs locally via Miniflare.

```bash
# Install pnpm
npm install -g pnpm

# Or enable via corepack
corepack enable
```

## Cloudflare Free Tier Limits

Keyflare works great on the free tier:

| Resource        | Free Tier Limit | Notes                        |
| --------------- | --------------- | ---------------------------- |
| Worker requests | 100,000/day     | Per-request billing on paid  |
| D1 storage      | 5 GB            | Per database                 |
| D1 rows read    | 5,000,000/day   |                              |
| D1 rows written | 100,000/day     |                              |
| Worker CPU time | 10ms            | Crypto ops are fast (\< 1ms) |

<Note>
  For most teams, the free tier is more than sufficient. Paid plans offer higher limits and additional features.
</Note>

## CLI Configuration

After deployment, the CLI stores configuration in:

```text
~/.config/keyflare/
├── config.yaml      # Default settings
└── credentials      # API key (file permissions 0600)
```

**config.yaml:**

```yaml
api_url: "https://keyflare.<account>.workers.dev"
project: "my-api"          # Default project (optional)
environment: "development"  # Default environment (optional)
```

**credentials:**

```text
kfl_user_a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6
```

## Environment Variables

Override CLI configuration with environment variables:

| Variable               | Description                                     |
| ---------------------- | ----------------------------------------------- |
| `KEYFLARE_API_KEY`     | API key (overrides credentials file)            |
| `KEYFLARE_API_URL`     | API URL (overrides config file)                 |
| `KEYFLARE_LOCAL`       | Set to `true` to target `http://localhost:8787` |
| `KEYFLARE_PROJECT`     | Default project (overrides config file)         |
| `KEYFLARE_ENV`        | Default environment (overrides config file)     |
| `CLOUDFLARE_API_TOKEN` | Used by `kfl init` (skips auth prompt)          |
| `DEBUG`                | Set to `keyflare` for CLI debug logs            |
